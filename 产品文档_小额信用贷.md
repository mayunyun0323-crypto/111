# 虚拟货币小额信用贷产品文档

## 1. 产品概述

### 1.1 产品名称
**CryptoCredit** - 加密货币小额信用贷

### 1.2 产品定位
面向加密货币用户的无抵押小额信用贷款服务，通过链上信用评估体系，为用户提供快速、便捷的借款体验。

### 1.3 核心价值
- **无需抵押**：打破传统 DeFi 超额抵押模式，用户无需锁定资产即可借款
- **快速放款**：审核通过后即时到账
- **灵活还款**：支持多种还款方式和周期

---

## 2. 目标用户

### 2.1 用户画像
| 特征 | 描述 |
|------|------|
| 年龄 | 18-45岁 |
| 身份 | 加密货币持有者、DeFi 活跃用户 |
| 需求 | 短期资金周转、避免卖出持仓 |
| 特点 | 有一定链上交易历史和信用记录 |

### 2.2 使用场景
- 短期资金周转，不想卖出持有的加密资产
- 参与新项目/IDO 需要额外资金
- 紧急支付需求

## 3. 产品功能

### 3.1 核心功能模块

#### 3.1.1 钱包连接
- 支持主流钱包连接（MetaMask、WalletConnect、Coinbase Wallet 等）
- 一键授权登录

#### 3.1.2 信用评估
基于用户链上数据进行信用评分：

| 评估维度 | 权重 | 说明 |
|----------|------|------|
| 钱包历史 | 20% | 钱包创建时间、活跃天数 |
| 交易记录 | 25% | 交易频率、交易金额 |
| 资产持有 | 25% | 历史资产持有量、持有时长 |
| DeFi 参与度 | 15% | 参与过的 DeFi 协议数量和深度 |
| 历史还款 | 15% | 在本平台或其他协议的还款记录 |

**信用等级划分：**
- **A级**（750-1000分）：最高额度 5000 USDT，利率 8%/年
- **B级**（600-749分）：最高额度 2000 USDT，利率 12%/年
- **C级**（450-599分）：最高额度 500 USDT，利率 18%/年
- **D级**（<450分）：暂不支持借款

#### 3.1.3 借款申请
**借款参数：**
| 参数 | 范围 |
|------|------|
| 借款金额 | 100 - 5000 USDT |
| 借款期限 | 7天 / 14天 / 30天 / 90天 |
| 支持币种 | USDT、USDC、DAI |
| 放款方式 | 直接转入用户钱包 |

**申请流程：**
1. 连接钱包
2. 系统自动评估信用分
3. 选择借款金额和期限
4. 确认借款条款
5. 签名确认
6. 资金到账

#### 3.1.4 还款管理
- **主动还款**：用户可随时提前还款，无罚息
- **自动还款**：授权后系统到期自动扣款
- **还款提醒**：到期前 3天、1天、当天推送提醒

#### 3.1.5 逾期管理

**逾期状态定义：**
| 状态 | 定义 | 标识颜色 |
|------|------|----------|
| 正常 | 未到还款日 | 青色 |
| 即将到期 | 距还款日 ≤3 天 | 橙色 |
| 已逾期 | 超过还款日 | 红色 |

**逾期处罚机制：**
| 逾期阶段 | 天数范围 | 滞纳金 | 信用影响 | 其他处罚 |
|----------|----------|--------|----------|----------|
| 轻度逾期 | 1-3天 | 每日 0.05% | 信用分 -10 | 无 |
| 中度逾期 | 4-7天 | 每日 0.1% | 信用分 -30 | 降低借款额度 |
| 重度逾期 | 8-14天 | 每日 0.15% | 信用分 -50 | 暂停借款资格 |
| 严重逾期 | 15-30天 | 每日 0.2% | 信用分 -100 | 列入灰名单 |
| 违约 | >30天 | 停止计息 | 信用归零 | 列入黑名单，启动催收 |

**逾期还款金额计算：**
```
应还总额 = 借款本金 + 借款利息 + 滞纳金
滞纳金 = 借款本金 × 日滞纳金率 × 逾期天数
```

**逾期处理流程：**
1. 到期日当天：系统推送还款提醒
2. 逾期第1天：发送逾期通知，开始计算滞纳金
3. 逾期第3天：再次提醒，提示信用影响
4. 逾期第7天：发送警告通知，暂停新借款
5. 逾期第14天：发送催收通知
6. 逾期第30天：标记为违约，列入黑名单

#### 3.1.6 黑名单机制

**黑名单触发条件：**
- 单笔借款逾期超过 30 天
- 多次恶意逾期（累计 3 次以上中度逾期）
- 检测到欺诈行为

**黑名单状态：**
| 属性 | 值 |
|------|-----|
| 信用评分 | 0 分 |
| 信用等级 | 黑名单 |
| 借款权限 | 禁止 |
| 还款权限 | 允许（必须还清） |

**黑名单解除条件：**
1. 还清所有逾期借款（本金 + 利息 + 滞纳金）
2. 等待冷静期（30 天）
3. 信用分从 300 分重新开始累积

**各页面黑名单展示规则：**

| 页面 | 展示内容 |
|------|----------|
| 首页 | 顶部显示红色警告横幅，"立即借款"按钮变灰禁用 |
| 借款页面 | 全屏黑名单提示，显示原因和解除方法，禁止借款操作 |
| 个人中心 | 信用评分显示为 0，等级显示"黑名单"，显示待还款项和解除指引 |

#### 3.1.7 还款激励排放 (Repay Emission)

**背景与目标：**

为了提升信用贷产品的资金回流效率及用户活跃度，我们将引入一套**常态化**的"还款激励排放机制"（Incentive Emission Mechanism）。该机制非短期营销活动，而是产品的基础功能之一，通过向按时还款的用户分配 LISTA 代币激励，促进生态健康发展。

| 目标 | 说明 |
|------|------|
| 促进回款 | 通过代币激励鼓励用户主动还款，减少坏账风险 |
| 培养习惯 | 通过高频（每日）和周期性（每周）的激励排放，引导用户形成"借款-还款-再借款"的良性循环 |
| 提升留存 | 通过"领取激励"和"查看激励记录"增加用户访问产品的频率 |

**参与资格 (Eligibility)：**
- 当日有"有效还款"，即获得激励排放资格
- 每人每日限 1 次
- 有效还款定义：需要支付利息的还款行为

**激励排放类型：**

| 类型 | 每日激励 (Daily Emission) | 每周激励 (Weekly Emission) |
|------|--------------------------|---------------------------|
| 周期 | 每日 00:00:00 - 23:59:59 (UTC) | 周一 00:00:00 - 周日 23:59:59 (UTC) |
| 条件 | 在该自然日内完成至少一次"有效还款" | 在该自然周内任意一天有过"有效还款"记录 |
| 排放额度 | 1,000 LISTA（固定额度） | 10,000 LISTA（固定额度） |
| 分配时间 | 次日 08:00 (UTC) 自动分配 | 下周一 08:00 (UTC) 自动分配 |
| 获得人数 | 1 人（从符合条件的地址中随机选取） | 1 人（从符合条件的地址中随机选取） |

**领取资格：**
- 获得激励的用户不能有逾期未还款项
- 如分配时存在逾期，将取消激励资格，激励顺延至下一位符合条件的用户

**激励价值测算：**

| 激励类型 | 最低利息 | 排放价值 | 参与地址阈值 | 说明 |
|----------|----------|----------|--------------|------|
| 每日激励 | 1.5 USDT/笔 | ~150 USDT (1,000 LISTA) | 100 个地址 | 还款地址 ≤100，用户预期收益为正 |
| 每周激励 | 1.5 USDT/笔 | ~1,500 USDT (10,000 LISTA) | 1,000 个地址 | 每周还款地址 ≤1,000，用户预期收益为正 |
